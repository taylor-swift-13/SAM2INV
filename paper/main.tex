%% Abstract
\begin{abstract}
Loop invariant generation is a central hard problem in program
verification.
Existing LLM-based approaches rely on extensive SMT solving, complex
workflows, and the most powerful closed-source models, yet still
struggle with this task---which demands logical reasoning, program
comprehension, and mastery of formal annotation syntax.
We present \textsc{SAM2INV}, a system that trains a small, locally
deployable model to generate high-quality loop invariants through
offline knowledge distillation with verifier feedback.
To overcome training-data scarcity, we first design
\textsc{LoopFactory}, a probabilistic DSL constrained by semantic
templates that synthesises structured numeric loop programs at
arbitrary scale.
We then build a three-stage sampling-and-filtering pipeline: $N$
parallel LLM calls are unioned into a single candidate pool and
filtered through (1)~an ACSL syntax filter, (2)~an execution-trace
semantic filter, and (3)~Houdini formal pruning via Frama-C/WP,
producing certified invariant annotations.
Rather than applying reinforcement learning---whose binary verifier
reward is too sparse, reward-hackable, and ineffective for weak base
models---we adopt a distillation strategy: a lightweight teacher
model explores multiple candidates per program during data
preparation; the verification pipeline filters them into chosen and
stage-labelled rejected examples; and the student is fine-tuned via
SFT followed by DPO, transferring the teacher's exploration benefit
with no human annotation or online reward model.
\end{abstract}
