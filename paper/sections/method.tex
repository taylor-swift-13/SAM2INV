%% ============================================================
\section{Method}\label{sec:method}
%% ============================================================

We now describe each component of \textsc{SAM2INV} in detail,
following the pipeline order introduced in Section~\ref{sec:overview}.

%% ------------------------------------------------------------
\subsection{Smart Dynamic Sampling}\label{sec:sampling}
%% ------------------------------------------------------------

The purpose of dynamic sampling is to collect execution traces that
expose the mathematical relationships maintained by the loop.
Naive random sampling often produces inputs that are either too
simple (all zeros, which collapse polynomial terms) or too large
(causing integer overflow or time-outs).
We therefore adopt a \emph{tiered} sampling strategy: parameter
values are partitioned into four tiers---special corner values
$\{0,\pm1\}$, small positives, medium values, and large
values---and input tuples are generated in phases of increasing
tier complexity, with a biased random fill for any remaining quota.
Each input is executed, and the variable state at every loop iteration
is recorded.
To keep prompts within LLM context limits, we retain only the first
and last $k$ iterations per run (default $k{=}3$) and group traces
into at most $G$ groups (default $G{=}10$).
Full details of the tier definitions, phased Cartesian sampling
algorithm, and trace formatting are given in
Appendix~\ref{app:sampling}.

%% ------------------------------------------------------------
\subsection{Parallel LLM Generation}\label{sec:generation}
%% ------------------------------------------------------------

Each LLM prompt is assembled from three components: a \emph{system
prompt} encoding the task definition and allowed ACSL constructs, the
\emph{source code} of the target function, and \emph{execution traces}
formatted as structured variable--value tables.
The system prompt encodes an \emph{assertion-driven} strategy:
decompose each postcondition conjunct as an invariant candidate,
weaken the loop guard to obtain a boundary invariant, and preserve
unmodified function parameters via \lstinline|\at(v, Pre)|.
Full prompts are reproduced in Appendix~\ref{app:prompts}.

To maximise the probability of finding the complete invariant set, we
issue $N$ parallel LLM calls (default $N{=}5$) with temperatures
sampled from $\{0.8, 1.0, 1.1, 1.2\}$.
Each call $j$ produces a candidate set $C_j$; these sets are
\emph{unioned}, not voted upon:
\[
C_{\text{pool}} = \bigcup_{j=1}^{N} C_j \quad
(\text{whitespace-normalised deduplication}).
\]
The union-then-filter design is motivated by a structural property
of loop invariants: the correct annotation is a \emph{conjunction}
of multiple clauses, and different clauses tend to be discovered by
different calls.
After filtering, the certified output $C^*$ is a post-hoc collective
result, not a selection from any individual response---a
\emph{non-attributability} property formalised in
Appendix~\ref{app:nonattrib}.

%% ------------------------------------------------------------
\subsection{Three-Stage Filter Pipeline}\label{sec:filtering}
%% ------------------------------------------------------------

The candidate pool is filtered through three progressively more
expensive stages; the design principle is to discard as many incorrect
candidates as cheaply as possible before invoking the next stage.

\paragraph{Stage~1: ACSL Syntax Filter.}
Each candidate is checked against structural ACSL rules---no
unsupported quantifiers, no forbidden operators, correct use of
\lstinline|\at| expressions, balanced brackets---and discarded
immediately on any violation.
This filter runs in microseconds per candidate.
The full rule set is listed in Appendix~\ref{app:syntax-gate}.

\paragraph{Stage~2: Execution-Trace Semantic Filter.}
Each surviving candidate is evaluated as a boolean expression over
every recorded loop state.
Any candidate falsified by at least one observed state is discarded
without invoking Frama-C.
This eliminates candidates with grossly incorrect arithmetic
relationships cheaply, though it cannot guarantee inductiveness
since the traces are finite.

\paragraph{Stage~3: Houdini Formal Pruning.}
The pre-screened candidates are submitted to Frama-C/WP.
We apply the Houdini algorithm~\cite{flanagan2001houdini}:
starting from the full candidate set, each iteration removes every
candidate whose verification condition fails, repeating until no
candidate is removed or the set is empty.

\begin{theorem}[Termination]\label{thm:termination}
Houdini pruning terminates in at most $|C|$ iterations,
since each iteration removes at least one candidate.
\end{theorem}

The output $C^*$ is the maximal inductive subset of the pre-screened
candidates.
Every invariant in $C^*$ is formally certified by Frama-C/WP.
If $C^*$ is non-empty but insufficient to prove the postcondition,
the system may optionally enter a brief repair loop (at most
$K{=}3$ iterations) feeding verification errors back to the LLM.
The complete per-loop algorithm is given in
Appendix~\ref{app:algorithm}.

Both improvements---union assembly and lightweight pre-screening---are
\emph{intrinsic to the pipeline itself}: they hold for any base LLM,
before any fine-tuning.
The offline knowledge distillation described next is a separate
improvement layer built on top of this already-strong baseline.

%% ------------------------------------------------------------
\subsection{Offline Knowledge Distillation}\label{sec:offline}
%% ------------------------------------------------------------

\paragraph{Why not reinforcement learning.}
Online RL with binary verifier feedback faces four structural
obstacles in this domain:
sparse reward (a weak model rarely produces a correct annotation
within a practical rollout budget);
reward hacking (trivial annotations such as
\lstinline|loop invariant true;| formally pass weak postconditions);
human-prior bias (a hand-crafted or trained reward model scales
poorly);
and non-attributability (the correct conjunction is assembled
post-hoc from $N$ calls, so per-call binary scoring mislabels
essential contributions;
see Appendix~\ref{app:nonattrib}).

\paragraph{Knowledge distillation via verifier feedback.}
We instead adopt offline \emph{knowledge distillation} from a
teacher model, but depart from standard distillation in a key
respect: rather than imitating the teacher's raw outputs, we move
GRPO-style exploration offline and embed the formal verifier's
signal into data preparation.
The teacher is a capable yet lightweight model---chosen so that the
student--teacher gap remains bridgeable and per-program token cost
stays tractable.

For each training program, the teacher generates $N$ candidate
annotations at diverse temperatures; all outputs are pooled and
passed through the three-stage filter.
The surviving certified set $C^*$ is the \emph{chosen} response.
Every candidate eliminated at stage $k \in \{1,2,3\}$ becomes a
\emph{rejected} response paired with $C^*$ to form a DPO record,
with stage-dependent loss weight ($w_1 \le w_2 \le w_3$) reflecting
that later-stage rejections are harder and more informative contrasts.
The SFT record is generated alongside each successful run.
The preference labels are ground-truth correct by construction:
no human annotation or trained reward model is needed.
Formal definitions and the stage-stratified loss are given in
Appendix~\ref{app:dpo}.

\paragraph{Two-stage fine-tuning.}
The student is first trained with SFT on chosen examples, then
aligned with DPO on preference pairs.
This two-stage recipe transfers the teacher's exploration benefit
to the student without online rollouts.
The filter pipeline certifies every output regardless of model
quality---a weaker model yields a smaller $C^*$ but never an
uncertified one---so the system remains sound throughout the
improvement cycle.
