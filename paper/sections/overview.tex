%% ============================================================
\section{System Overview}\label{sec:overview}
%% ============================================================

\sam{} is built around a single unified pipeline
(Figure~\ref{fig:pipeline}) that serves \emph{both} training-data
synthesis and test-time inference.
Given a C function with a \texttt{while} loop, precondition, and
postcondition, the pipeline outputs ACSL-annotated code verified by
Frama-C/WP.
During training data preparation it additionally produces SFT and DPO
records; at inference time those stages are simply deactivated.
Table~\ref{tab:mode-diff} summarises which components are active in
each mode.

\item \textbf{ACSL Syntax Filter} --- Reject
      structurally invalid conjuncts.
\item \textbf{Trace Semantic Filter} --- Discard
      conjuncts falsified by observed states.
\item \textbf{Semantic Deduplication} --- Collapse
      logically equivalent conjuncts via auxiliary LLM.
\item \textbf{Houdini Formal Pruning} --- Iteratively
      remove non-inductive conjuncts via Frama-C/WP.
\item \textbf{Trivial Invariant Filter} --- Exclude
      formally correct but semantically vacuous invariants.
\item \textbf{Reverse CoT \& Distillation} ---
      Synthesise reasoning chains; construct chosen/rejected pairs;
      augment and train via SFT\,+\,DPO.
\end{enumerate}

\paragraph{Phase~I: Generation (\ding{182}--\ding{183}).}
The program is first executed under tiered inputs to collect
execution traces $\mathcal{T}$.
The value domain is partitioned into four tiers of increasing
magnitude---corner values ($0, \pm1$), small positives ($2$--$5$),
medium values ($6$--$50$), and large values ($51$--$500$)---and
inputs are drawn in phases of increasing tier complexity to cover
both boundary and interior behaviour.
To fit LLM context limits, each trace retains the first and last $k$
iterations (default $k{=}3$); traces are grouped into at most $G$
groups (default $G{=}10$).
A structured \emph{loop context} is then assembled from the traces,
a static-analysis record (loop guard, variable scopes, pre-condition),
and a symbolically derived transition relation
(Section~\ref{sec:loop-context}).
$N$ parallel LLM calls at diverse temperatures each return a CoT
reasoning trace and candidate invariants; all invariant expressions
are decomposed into conjuncts and unioned into a single pool
$C_{\mathrm{pool}}$.
During training, the teacher $\mathcal{M}_T$ is called with a large
$N$ to maximise conjunct coverage; at inference, the student
$\mathcal{M}_S$ is called with a smaller $N$.

\paragraph{Phase~II: Multi-Stage Filtering (\ding{184}--\ding{188}).}
Five filters of increasing cost refine the pool to a certified,
non-trivial invariant set $C^*$ (Section~\ref{sec:filtering}).
Every surviving conjunct is formally verified by Frama-C/WP.
This phase is identical in both modes.

\paragraph{Phase~III: Distillation (\ding{189}).}
Active only during training.
A reverse CoT model synthesises a reasoning chain for $C^*$.
The result forms the \emph{chosen} sample; rejected candidates
become DPO negatives (Section~\ref{sec:offline}).
Tiered sampling details are in Appendix~\ref{app:sampling}.
